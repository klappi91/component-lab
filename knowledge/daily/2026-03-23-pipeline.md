## Prozess-Experiment: Designer-Agent → Builder Pipeline

### Was wurde getestet
Erstes Prozess-Experiment: Ein spezialisierter Designer-Agent entwirft das Konzept, ein Builder-Agent setzt es um, ein Asset-Generator generiert Bilder — drei spezialisierte Rollen statt ein Generalist.

### Pipeline
1. **Designer-Agent** (Opus, Sub-Agent, ~5 Min) → Vollstaendiges Konzept "Warmer Brutalismus trifft Editorial Craft"
2. **Asset-Generator** (Sonnet, Background-Agent, ~5 Min) → 6 Bilder (3 Service, 3 Mockup)
3. **Builder-Agent** (Opus, tmux-Session, ~10 Min) → Komplette Homepage, 14 Dateien, 8 Sektionen
4. **Orchestrator** (ich) → QA, Fixes (Fonts + Bilder-Integration), Deploy

### Ergebnis
- **Website deployed:** https://exp-warm-brutal-rcyrwlwgq-christian-klapproths-projects.vercel.app
- **GitHub:** https://github.com/klappi91/weblab-exp-warm-brutal
- **Build:** Fehlerfrei (0 TypeScript-Fehler)
- **Gesamtzeit:** ~25 Min (Designer 5 + Builder 10 + QA/Fix 10)

### Was funktioniert hat
- **Designer-Konzept-Qualitaet: EXZELLENT** — detailliert, mutig, konkreter Content, Asset-Prompts mitgeliefert, klare Signature Moments, 7-Akt-Narrative
- **Asset-Qualitaet: GUT** — gemini-image liefert brauchbare Editorial-Fotos und Mockups
- **Builder-Geschwindigkeit: BEEINDRUCKEND** — 10 Min fuer 14 Dateien, 8 Sektionen, Canvas-Animation, GSAP, Lenis, Custom Cursor
- **Parallele Ausfuehrung** — Assets im Hintergrund waehrend Builder laeuft
- **Farb/Font-System** — Builder hat Design-System 1:1 umgesetzt (Burnt Vermillion genau 3x)

### Was NICHT funktioniert hat
- **Builder nutzte generierte Bilder nicht** — CSS-Gradients statt der generierten Assets. Musste manuell fixen.
- **Fontshare-Fonts** — CSS @import wurde im Build nicht korrekt aufgeloest. Fix: <link> Tags im <head>
- **Canvas-Animation** — im Headless-Browser unsichtbar. Muss in echtem Browser getestet werden.
- **Horizontal Overflow** — 40px in Arsenal-Sektion auf Desktop (overflow-x:hidden versteckt es)
- **Builder kennt Assets nicht** — Er wusste nicht welche Bilder vorhanden sind, weil sie parallel generiert wurden

### Erkenntnisse (fuer Constitution)

1. **Designer→Builder Pipeline funktioniert grundsaetzlich** — klares Konzept = schnellerer, fehlerfreierer Build
2. **Asset-Pipeline braucht Sync** — Builder muss wissen welche Assets vorhanden sind. Entweder: Assets ZUERST, dann Builder. Oder: Asset-Manifest das der Builder referenziert.
3. **Fontshare in Next.js** — CSS @import funktioniert nicht zuverlaessig im Build. Immer <link> Tags im layout.tsx verwenden.
4. **Builder folgt Konzept gut** — Sektions-Struktur, Farben, Typografie wurden 1:1 umgesetzt. Nur bei Assets hat er geguesst.
5. **Gesamtzeit ~25 Min** — akzeptabel fuer eine vollstaendige Website mit 8 Sektionen und 6 generierten Bildern.
